Temperature and Humidity Report

PILOT Extension Generates a temperature and humidity readings report for selected vehicles over a chosen time range.

Purpose

Adds a navigation tab to PILOT for building temperature and humidity reports across the fleet. The operator picks any number of vehicles, sets a From / To time range, and exports the resulting sensor readings to the main panel or to CSV.

Features

  • Navigation tab with vehicle list and checkbox selection.
  • Select all / clear all helpers, and one-click reload from PILOT.
  • From / To date-time pickers (uses Pilot.utils.DateTimeField when available).
  • Report grid with vehicle, time, temperature (°C) and humidity (%).
  • One-click CSV export of the visible report.
  • Localized strings through l(...).

Data Sources

  • Vehicles: /ax/tree.php with vehs=1, state=1. Hierarchical groups are flattened; the parent group name is shown next to each vehicle.
  • Report: /ax/report.php called with type=temp_humidity, comma-separated agents, and Unix-second from / to parameters. The response is expected to be an array (or {rows: [...]}) of objects with agentid, ts, temp, humidity.

Setup

  1. Upload Module.js, style.css, and doc/index.html to your extension host.
  2. Register the extension Module.js URL inside PILOT.
  3. Reload PILOT — the Temp/Humidity tab appears in the left navigation.
  4. Open the tab, tick vehicles, choose From / To, press Generate Report.

Expected Report Response

{
  "rows": [
    { "agentid": 101, "ts": 1715600000, "temp": 4.2, "humidity": 71.3 },
    { "agentid": 101, "ts": 1715603600, "temp": 4.4, "humidity": 70.8 },
    { "agentid": 204, "ts": 1715600000, "temp": -18.6, "humidity": 55.0 }
  ]
}

Limitations

  • Requires that the host PILOT build exposes a temperature/humidity report endpoint under /ax/report.php. If your installation uses a different path or payload format, adjust the URL/params in MainPanel.loadReport and the field normalization there.
  • Requires Pilot.utils.LeftBarPanel in the runtime for the styled navigation tab; falls back to Ext.panel.Panel if it is missing.
  • CSV export runs in the browser (Blob download) and is not pushed back to PILOT.
