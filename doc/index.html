Temperature and Humidity Report V222

PILOT Extension Generates a temperature and humidity readings report for selected vehicles over a chosen time range.

Purpose

Adds a navigation tab to PILOT — its icon sits at the top of the side rail, in the same style as the planets example. The operator picks any number of vehicles, sets a From / To time range, and the paired main panel fills with temperature and humidity readings. The result can also be rendered as a chart or exported to CSV.

Features

  • Top-aligned icon tab in the left navigation rail (iconAlign: 'top').
  • Vehicle list with multi-select checkboxes (header checkbox = select all).
  • Select all / Clear / Reload helpers and one-click refresh.
  • From / To pickers — Pilot.utils.DateTimeField when available, falls back to datefield.
  • Report grid grouped by vehicle: time, temperature (°C), humidity (%).
  • Optional Highcharts viewer with dual Y axes (temperature / humidity).
  • One-click CSV export of the visible report.
  • Strings localized through l(...).

Data Sources

  • Vehicles: read directly from the in-memory online tree skeleton.navigation.online.online_tree.store using cascadeBy (matches communal/Tree.js). Falls back to /ax/tree.php?vehs=1&state=1 when the tree is unavailable; the hierarchical children payload is walked recursively as AI_SPECS §5 requires.
  • Report: POST /ax/report.php with type=temp_humidity, comma-separated agents, and Unix-second from / to parameters. The endpoint is configurable via reportUrl / reportType on the Map class.

Expected Report Response

{
  "rows": [
    { "agentid": 101, "ts": 1715600000, "temp":   4.2, "humidity": 71.3 },
    { "agentid": 101, "ts": 1715603600, "temp":   4.4, "humidity": 70.8 },
    { "agentid": 204, "ts": 1715600000, "temp": -18.6, "humidity": 55.0 }
  ]
}

The normalizer also accepts temperature/hum/t1/h1 and either {rows: [...]}, {data: [...]}, or a bare array.

Setup

  1. Upload Module.js, Tab.js, Map.js, style.css, and doc/index.html to your extension host.
  2. Register the extension's Module.js URL inside PILOT.
  3. Reload PILOT — the Temp/Humidity tab appears in the left navigation, icon on top.
  4. Open the tab, tick vehicles, choose From / To, press Generate Report.

Limitations

  • Requires that the host PILOT build exposes a temperature/humidity report endpoint reachable at reportUrl. If your installation uses a different sensor-history path or payload, override reportUrl / reportType or adjust the normalization block in Map.js → loadReport.
  • Falls back to Ext.panel.Panel if Pilot.utils.LeftBarPanel is missing in the target runtime (icon-on-top look depends on the host build providing that class).
  • Chart view requires window.Highcharts; otherwise the grid and CSV export still work.
  • CSV export runs in the browser (Blob download) and is not pushed back to PILOT.
